## Contract Architecture

### The Profile → Intent Boundary

```
┌─────────────────────┐         ┌──────────────────────────┐
│  GrooveProfileV1    │────────▶│  GrooveControlIntentV1   │
│  (Persistent)       │         │  (Ephemeral)             │
│                     │         │                          │
│  "What kind of      │         │  "What should the        │
│   groove player     │         │   system do RIGHT NOW?"  │
│   is this?"         │         │                          │
└─────────────────────┘         └──────────────────────────┘
        ▲                                   │
        │                                   ▼
   Groove Layer                    ┌─────────────────┐
   (Analyzer)                      │  MidiControlPlan │
                                   │  (zt-band)       │
                                   └─────────────────┘
```

### Key Design Properties

| Property | Implementation |
|----------|----------------|
| **No learning state leaks** | Intent is ephemeral (horizon_ms bounds validity) |
| **Latency-safe** | horizon_ms: 50–60000ms |
| **Testable** | Pure input/output contracts |
| **Composable** | zt-band, Coach, Arranger all consume Intent |
| **Upgradeable** | `extensions` namespace for additive fields |

## zt-band Adapter Pipeline

### Intent → MIDI Control Plan

```python
from zt_band.adapters import build_midi_control_plan

intent = load_intent()  # GrooveControlIntentV1 dict
plan = build_midi_control_plan(intent)

# plan.clock_mode      → "midi_clock_master" | "midi_clock_follow"
# plan.target_bpm      → 92.0
# plan.humanize_ms     → 7.5 (derived from expression_window * (1 - lock_strength))
# plan.global_microshift_ms → -18.0 (clamped to ±30ms)
# plan.cc_messages     → [(0xB0, 20, 95), (0xB0, 21, 32), ...]
```

### MIDI CC Mapping (default_map.py)

| CC | Parameter | Range |
|----|-----------|-------|
| CC20 | Tightness (lock_strength) | 0–127 |
| CC21 | Humanize | 0–127 |
| CC22 | Assist Gain | 0–127 |
| CC23 | Expression Window | 0–127 |
| CC24 | Drift Correction | 0/64/127 (none/soft/aggressive) |
| CC25 | Recovery Enable | 0/127 (off/on) |

### Clock Mode Selection

| control_modes contains | clock_mode |
|------------------------|------------|
| `follow`, `assist` | `midi_clock_follow` |
| `stabilize`, `challenge`, `recover` | `midi_clock_master` |

## MIDI Clock Master

```python
from zt_band.midi import MidiClockMaster

def my_sender(data: bytes) -> None:
    midi_port.send(data)

clock = MidiClockMaster(send_bytes=my_sender)
clock.start(bpm=120.0)

while running:
    clock.set_bpm_target(plan.target_bpm)  # From Groove Control Intent
    clock.tick()                           # Call every 1-5ms
    time.sleep(0.002)

clock.stop()
```

### TempoSmoother Behavior

- **tau_s=0.25**: Exponential smoothing time constant
- **max_slew_bpm_per_s=50.0**: Hard rate limit
- **Bounded catch-up**: max_catchup_ticks=6 (prevents runaway if loop overruns)

## Code Generation Patterns

### When creating new contracts

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "$id": "sg-ai:contracts/<name>_v1.schema.json",
  "additionalProperties": false,
  "required": ["schema_id", "schema_version", ...],
  "properties": {
    "schema_id": { "type": "string", "const": "<name>" },
    "schema_version": { "type": "string", "const": "v1" },
    "extensions": {
      "type": "object",
      "additionalProperties": true,
      "description": "Forward-compatible extension space."
    }
  }
}
```

### When adding zt-band adapters

```python
# Location: src/zt_band/adapters/<name>.py
# Import pattern:
from zt_band.adapters.midi_control_plan import MidiControlPlan
from zt_band.adapters.default_map import DEFAULT_MAP
```

### When adding tests

```python
# Location: tests/test_<module>.py
# Import from installed package:
from zt_band.adapters import build_midi_control_plan
from zt_band.midi import MidiClockMaster
```

## Terminology (Frozen)

| Term | Definition |
|------|------------|
| **Groove Profile** | Persistent rhythmic personality traits (HOW player grooves) |
| **Control Intent** | Ephemeral prescriptive output (WHAT system does next) |
| **horizon_ms** | Validity window for intent before regeneration |
| **lock_strength** | 0=free follow, 1=hard lock to grid |
| **control_modes** | Active strategies: follow, assist, stabilize, challenge, recover |
| **drift_correction** | none, soft, aggressive — tempo drift handling |
