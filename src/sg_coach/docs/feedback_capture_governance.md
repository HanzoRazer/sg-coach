# Feedback Capture Governance

Sprint 5 Dev Order 2: Capture contract only, no storage or learning.

## Purpose

Define how UserFeedbackEvent objects are created, validated, and captured after a user responds to a CoachFinding or ActionRecommendationSet.

## Core Principle

```
Capture creates one durable UserFeedbackEvent.
It does not infer learning yet.
```

## Ownership Boundaries

| Component | Owner | Responsibility |
|-----------|-------|----------------|
| FeedbackCaptureRequest schema | sg-spec | Capture request contract |
| UserFeedbackEvent schema | sg-spec | Stored event contract |
| capture_feedback() | sg-coach | Request → Event conversion |
| validate_feedback_linkage() | sg-coach | Linkage validation |
| Event storage | sg-agentd (future) | Persistence |
| Learning derivation | sg-coach (future) | LearningSignal computation |

## Capture Flow

```
User Action
    ↓
FeedbackCaptureRequest
    ↓
capture_feedback()
    ↓
UserFeedbackEvent (id auto-generated)
    ↓
[Future: sg-agentd stores event]
    ↓
[Future: sg-coach derives LearningSignal]
```

## Field Semantics

### FeedbackCaptureRequest

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| session_id | str | No | Links to practice session |
| finding_id | str | No | Links to CoachFinding |
| recommendation_id | str | No | Links to ActionRecommendationSet |
| response_type | UserFeedbackResponseType | **Yes** | User's response |
| confidence | float | No | User's confidence (0.0-1.0) |
| comment | str | No | User's explanation (max 500) |
| corrected_result | dict | No | Structured correction data |
| outcome | PracticeOutcome | No | Observable outcome |
| source | str | No | Capture source (ui, agentd, cli, test) |
| interaction_context | dict | No | Flexible context (defaults to {}) |

### UserFeedbackEvent

All fields from FeedbackCaptureRequest plus:

| Field | Type | Description |
|-------|------|-------------|
| id | str | Auto-generated "uf_<uuid>" |
| timestamp | datetime | When captured (auto-set) |

## Linkage Rules

### Recommended Linkage

At least one of:
- `session_id` — links to practice session
- `finding_id` — links to specific CoachFinding
- `recommendation_id` — links to specific ActionRecommendationSet

### Validation Policy

```python
warnings = validate_feedback_linkage(request)
```

Returns structured warnings:
```python
{
    "field": "session_id",
    "code": "missing_session_id", 
    "message": "Feedback event has no session_id"
}
```

**Missing linkage produces warnings, not failure.**

Rationale: v1 prioritizes capture over strict validation. Orphan feedback is valid but less useful.

## Append-Only Rule

```
Feedback events are APPEND-ONLY.
```

- Never mutate an existing UserFeedbackEvent
- Never delete feedback history
- Never update findings based on feedback (in capture phase)
- Never update recommendations based on feedback (in capture phase)

## What Capture Does NOT Do

| Does | Does NOT |
|------|----------|
| Create UserFeedbackEvent | Store event |
| Validate linkage (warnings) | Block on invalid linkage |
| Copy all request fields | Compute derived fields |
| Auto-generate event ID | Require external ID |
| Set timestamp | Compute LearningSignal |

## corrected_result Structure

The `corrected_result` field accepts flexible structured data:

```python
# Corrected note
{"corrected_note": "C#4", "original_note": "C4"}

# Corrected timing
{"corrected_beat": 2.5, "original_beat": 2.0, "unit": "beat"}

# User-marked span
{"start_bar": 3, "end_bar": 4, "issue": "fingering"}

# Perceived cause
{"cause": "string_buzz", "fret": 5, "string": 3}
```

**Must be structured data, not only prose.**

## Examples

### Basic Capture

```python
from sg_coach import capture_feedback
from sg_spec.schemas.user_feedback import (
    FeedbackCaptureRequest,
    UserFeedbackResponseType,
)

request = FeedbackCaptureRequest(
    session_id="sess_abc123",
    finding_id="find_xyz789",
    response_type=UserFeedbackResponseType.helped,
    confidence=0.9,
)

event = capture_feedback(request)
# event.id = "uf_<auto-generated>"
# event.timestamp = <current UTC>
```

### With Validation

```python
from sg_coach import capture_feedback, validate_feedback_linkage

request = FeedbackCaptureRequest(
    response_type=UserFeedbackResponseType.rejected,
    comment="The note was actually correct",
)

warnings = validate_feedback_linkage(request)
# warnings = [
#     FeedbackLinkageWarning(code="missing_session_id", ...),
#     FeedbackLinkageWarning(code="missing_target_linkage", ...),
# ]

# Capture proceeds despite warnings
event = capture_feedback(request)
```

### With Override

```python
from datetime import datetime, timezone

event = capture_feedback(
    request,
    event_id="uf_custom_id",
    timestamp=datetime(2026, 5, 4, 12, 0, 0, tzinfo=timezone.utc),
)
```

## Future Integration

### sg-agentd (Future)

```
Records UserFeedbackEvent when user provides feedback.
Stores events in append-only log.
Does NOT compute LearningSignal.
```

### sg-coach (Future)

```
Interprets feedback patterns.
Computes LearningSignal from events + outcomes.
Updates coaching based on accumulated signals.
```

### sg-curriculum (Future)

```
Uses LearningSignal to adjust drill assignments.
Personalizes difficulty based on feedback history.
```
